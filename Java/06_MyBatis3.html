第二章 MyBatis  项目配置

 

1、environments

MyBatis 支持多个环境，可以任意配置；

 

2、transactionManager

MyBatis 支持两种类型的事务管理器：JDBC 和 MANAGED(托管)；

1）JDBC：应用程序负责管理数据库连接的生命周期(Tomcat支持JDBC)；

2）MANAGED：由应用服务器负责管理数据库连接的生命周期；(一般商业服务器才有此功能，如JBOSS,WebLogic)

 

截取部分

3、dataSource

用来配置数据源；类型有：UNPOOLED，POOLED，JNDI；

1）UNPOOLED，传统方式，没有连接池，每次数据库操作，MyBatis 都会创建一个新的连接，用完后，关闭；适合小并发项目；

2）POOLED，用上了连接池（常用）；

3）JNDI，使用应用服务器配置 JNDI 数据源获取数据库连接；

 

4、properties

配置属性

 

5、typeAliases

给类的完成限定名取别名，方便使用；

 

6、mappers

4、Mybatis  缓存

Mybatis 默认情况下，MyBatis 启用一级缓存，即同一个 SqlSession 接口对象调用了相同的 select 语句，则直

接会从缓存中返回结果，而不是再查询一次数据库，针对单个用户？；

开发者可以自己配置二级缓存，二级缓存是全局的，针对所有用户？；

 

默认情况下，select 使用缓存的，insert update delete 是不使用缓存的；
