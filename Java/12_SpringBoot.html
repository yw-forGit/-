1、SpringBoot简介

 

Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。通过这种方式，Spring Boot致力于在蓬勃发展的快速应用开发领域(rapid application development)成为领导者。

 

特点

1. 创建独立的Spring应用程序

2. 嵌入的Tomcat，无需部署WAR文件

3. 简化Maven配置

4. 自动配置Spring

5. 提供生产就绪型功能，如指标，健康检查和外部配置

6. 绝对没有代码生成并且对XML也没有配置要求

1、Spring Data Jpa简介

JPA诞生（JDK 5.0）的缘由是为了整合第三方ORM框架，建立一种标准的方式，百度百科说是JDK为了实现ORM的天下归一，目前也是在按照这个方向发展，但是还没能完全实现。在ORM框架中，Hibernate是一支很大的部队，使用很广泛，也很方便，能力也很强，同时Hibernate也是和JPA整合的比较良好，我们可以认为JPA是标准，事实上也是，JPA几乎都是接口，实现都是Hibernate在做，宏观上面看，在JPA的统一之下Hibernate很良好的运行。

 

　　上面阐述了JPA和Hibernate的关系，那么Spring-data-jpa又是个什么东西呢？这地方需要稍微解释一下，我们做Java开发的都知道Spring的强大，到目前为止，企业级应用Spring几乎是无所不能，无所不在，已经是事实上的标准了，企业级应用不使用Spring的几乎没有，这样说没错吧。而Spring整合第三方框架的能力又很强，他要做的不仅仅是个最早的IOC容器这么简单一回事，现在Spring涉及的方面太广，主要是体现在和第三方工具的整合上。而在与第三方整合这方面，Spring做了持久化这一块的工作，我个人的感觉是Spring希望把持久化这块内容也拿下。于是就有了Spring-data-**这一系列包。包括，Spring-data-jpa,Spring-data-template,Spring-data-mongodb,Spring-data-redis，还有个民间产品，mybatis-spring，和前面类似，这是和mybatis整合的第三方包，这些都是干的持久化工具干的事儿。

 

　　这里介绍Spring-data-jpa，表示与jpa的整合。我们都知道，在使用持久化工具的时候，一般都有一个对象来操作数据库，在原生的Hibernate中叫做Session，在JPA中叫做EntityManager，在MyBatis中叫做SqlSession，通过这个对象来操作数据库。我们一般按照三层结构来看的话，Service层做业务逻辑处理，Dao层和数据库打交道，在Dao中，就存在着上面的对象。那么ORM框架本身提供的功能有什么呢？答案是基本的CRUD，所有的基础CRUD框架都提供，我们使用起来感觉很方便，很给力，但业务逻辑层面的处理，ORM框架是没有提供的，如果使用原生的框架，业务逻辑代码我们一般会自定义，会自己去写SQL语句，然后执行。在这个时候，Spring-data-jpa的威力就体现出来了，ORM框架提供的能力他都提供，ORM框架没有提供的业务逻辑功能Spring-data-jpa也提供，全方位的解决用户的需求。使用Spring-data-jpa进行开发的过程中，常用的功能，我们几乎不需要写一条sql语句，至少在我看来，企业级应用基本上可以不用写任何一条sql，当然spring-data-jpa也提供自己写sql的方式，这个就看个人怎么选择，都可以。

import org.springframework.web.bind.annotation.PostMapping;

import org.springframework.web.bind.annotation.RequestMapping;

import org.springframework.web.bind.annotation.RequestMethod;

import org.springframework.web.servlet.ModelAndView;

 

import com.jluzh.dao.BookDao;

import com.jluzh.entity.Book;

 

/**

 * 注意：想要返回请求转发、重新定向（视图不影响）等，是不能用RestController，如果某些请求需要返回单纯的字符串，加ResponseBody即可

@Controller

@RequestMapping("/book")

public class JpaCRUD {

  

   @Resource // 直接能注入

   private BookDao bookDao;

  

   /**

     * 查询图书

    * @return

    */

   @RequestMapping("/list")

   public ModelAndView list() {

      ModelAndView mav = new ModelAndView();

      // spring data jpa底层是用了hibernate，故这里的查找也类似于hibernate

      mav.addObject("bookList", bookDao.findAll());

      // 返回的视图是src/main/resources/templates下的ftl文件

      mav.setViewName("bookList");

      return mav;

   }

  

   /**

    * 添加图书

    * @return

    */

   @RequestMapping(value="/add", method=RequestMethod.POST) // post请求

   public String add(Book book) {

      bookDao.save(book);

      // 直接转发到/book/list请求（默认是项目根路径）

      return "forward:/book/list";

   }

  

   /**

    * 准备更新图书

    * @return

    */

   @RequestMapping(value="/preUpdate/{id}") // 从链接获取参数

   public ModelAndView preUpdate(@PathVariable("id") Integer id) {

      ModelAndView mav = new ModelAndView();

      // 回显数据，getOne就是getById

      mav.addObject("book", bookDao.getOne(id));

      mav.setViewName("preUpdate");

      return mav;

   }

  

   /**

    * 更新图书

    * @return

    */

   @PostMapping("/update") // @PostMapping就是post请求

   public String update(Book book) {

      // update一样是save

      bookDao.save(book);

      // 直接转发到/book/list请求（默认是项目根路径）

      return "forward:/book/list";

   }

  

   /**

    * 删除图书

    * @return

    */

   @GetMapping(value="/delete") // @GetMapping就是get请求

   public String delete(Integer id) {

      bookDao.deleteById(id);

      // 直接转发到/book/list请求（默认是项目根路径）

      return "forward:/book/list";

   }

}

           return name;

      }

      public String getAuthor() {

           return author;

package com.jluzh.controller;

 

import java.util.List;

 

import javax.annotation.Resource;

import javax.persistence.criteria.CriteriaBuilder;

import javax.persistence.criteria.CriteriaQuery;

import javax.persistence.criteria.Predicate;

import javax.persistence.criteria.Root;

 

import org.springframework.data.jpa.domain.Specification;

import org.springframework.data.jpa.repository.JpaSpecificationExecutor;

import org.springframework.stereotype.Controller;

import org.springframework.web.bind.annotation.RequestMapping;

import org.springframework.web.servlet.ModelAndView;

 

import com.jluzh.dao.BookDao;

import com.jluzh.entity.Book;

 

/**

 * 注意：想要返回请求转发、重新定向（视图不影响）等，是不能用RestController，如果某些请求需要返回单纯的字符串，加ResponseBody即可

 * @author WIN

 *

 */

@Controller

@RequestMapping("/book")

public class BookController {

    

     @Resource // 直接能注入

     private BookDao bookDao;

    

     /**

      * 根据名称查询（hql查询）

      * @return

      */

     @RequestMapping("/findByName")

     public ModelAndView findByName() {

         ModelAndView mav = new ModelAndView();

         List<Book> bookList = bookDao.findByName("编程");

         mav.addObject("bookList", bookList);

         // 返回的视图是src/main/resources/templates下的ftl文件

         mav.setViewName("bookList");

         return mav;

     }

    

     /**

      * 随机查询（sql查询）

      * @return

      */

     @RequestMapping("/randomList")

     public ModelAndView randomList() {

         ModelAndView mav = new ModelAndView();

         List<Book> bookList = bookDao.randomList(2);

         mav.addObject("bookList", bookList);

         // 返回的视图是src/main/resources/templates下的ftl文件

         mav.setViewName("bookList");

         return mav;

     }

    

     /**

      * 根据名称、作者查询（动态sql，BookDao接口只需再继承JpaSpecificationExecutor<Book>即可）

      * @return

      */

     @RequestMapping("/findByNameAuthor")

     public ModelAndView findByNameAuthor(Book book) {

         ModelAndView mav = new ModelAndView();

         // 使用该findAll，BookDao接口需要再继承JpaSpecificationExecutor<Book>（接口可以多继承）

         List<Book> bookList = bookDao.findAll(new Specification<Book>() {

              @Override

              public Predicate toPredicate(Root<Book> root, CriteriaQuery<?> query, CriteriaBuilder criteriaBuilder) {

                   Predicate predicate = criteriaBuilder.conjunction();

                   if(book != null) {

                       // 动态拼接sql

                       if(book.getName() != null && !"".equals(book.getName())) {

                            // like表示模糊查询，root.get("name")与"%" + book.getName() + "%"就是key-value的对应

                            predicate.getExpressions().add(criteriaBuilder.like(root.get("name"), "%" + book.getName() + "%"));

                       }

                       if(book.getAuthor() != null && !"".equals(book.getAuthor())) {

                            predicate.getExpressions().add(criteriaBuilder.like(root.get("author"), "%" + book.getAuthor() + "%"));

                       }

                   }

                  

                   // 记得把predicate返回，否则查询将没有上面的动态拼接条件，而是查询所有

                   return predicate;

              }

         });

         mav.addObject("bookList", bookList);

         // 返回的视图是src/main/resources/templates下的ftl文件

         mav.setViewName("bookList");

         return mav;

     }

}

 

 

BookDao.java

package com.jluzh.dao;

 

 

import java.util.List;

 

import org.springframework.data.jpa.repository.JpaRepository;

import org.springframework.data.jpa.repository.JpaSpecificationExecutor;

import org.springframework.data.jpa.repository.Query;

 

import com.jluzh.entity.Book;

 

/**

 * 接口dao，继承JpaRepository之后可以有很多类似于Hibernate操作数据库的方法

 * Book是实体类，Integer是主键类型

 * @author WIN

 *

 */

public interface BookDao extends JpaRepository<Book, Integer>, JpaSpecificationExecutor<Book> {

 

    // 这是hql的写法（这种注解方式类似于Mybatis的注解，这种dao也是类似于mybatis的dao）。?1代表方法里面的第一个参数

    @Query("select b from Book b where b.name like %?1%")  // 注意没有select * from的写法

    public List<Book> findByName(String name);

   

    // 这是原生sql（nativeQuery=true）。RAND是mysql的函数，这里是随机获取n条数据

    @Query(value="select * from book order by RAND() limit ?1", nativeQuery=true)

    public List<Book> randomList(Integer n);

   

}

5、接口规范方法名查询

JpaRepository支持接口规范方法名查询。意思是如果在接口中定义的查询方法符合它的命名规则，就可以不用写实现。

很简单，就是findBy + 字段名 + 关键字（即where条件），这个方法就可以直接查询，无需自己写sql实现。

第六章 SpringBoot之表单验证（@Valid）

 

SpringBoot提供了强大的表单验证功能实现，给我们省去了写验证的麻烦。

 

限制

说明

@Null

限制只能为null

@NotNull

限制必须不为null

@AssertFalse

限制必须为false

@AssertTrue

限制必须为true

@DecimalMax(value)

限制必须为一个不大于指定值的数字

@DecimalMin(value)

限制必须为一个不小于指定值的数字

@Digits(integer,fraction)

限制必须为一个小数，且整数部分的位数不能超过integer，小数部分的位数不能超过fraction

@Future

限制必须是一个将来的日期

@Max(value)

限制必须为一个不大于指定值的数字

@Min(value)

限制必须为一个不小于指定值的数字

@Past

限制必须是一个过去的日期

@Pattern(value)

限制必须符合指定的正则表达式

@Size(max,min)

限制字符长度必须在min到max之间

@Past

验证注解的元素值（日期类型）比当前时间早

@NotEmpty

验证注解的元素值不为null且不为空（字符串长度不为0、集合大小不为0）

@NotBlank

验证注解的元素值不为空（不为null、去除首位空格后长度为0），不同于@NotEmpty，@NotBlank只应用于字符串且在比较时会去除字符串的空格

@Email

验证注解的元素值是Email，也可以通过正则表达式和flag指定自定义的email格式

import org.springframework.web.bind.annotation.RequestMapping;

import org.springframework.web.bind.annotation.ResponseBody;

/**

 * 注意：想要返回请求转发、重新定向（视图不影响）等，是不能用RestController，如果某些请求需要返回单纯的字符串，加ResponseBody即可

 * @author WIN

 *

 */

@Controller

@RequestMapping("/student")

public class StudentController {

  

   @Resource

   private StudentService studentService;

  

   @RequestMapping("/add")

   @ResponseBody

   public String add(@Valid Student student, BindingResult br) {

      // 有错误，即注解的条件校验不成功

      if(br.hasErrors()) {

         // 这里获取的message就是注解里面的message

         return br.getFieldError().getDefaultMessage();

      }

      studentService.add(student);

      return "添加成功";

   }

}

第七章 SpringBoot之切面AOP

SpringBoot提供了强大AOP支持，我们前面讲解过AOP面向切面，所以这里具体AOP原理就补具体介绍：

AOP切面主要是切方法，我们一般搞一些日志分析和事务操作，要用到切面，类似拦截器：

 

@Aspect注解是切面注解类

@Pointcut切点定义

@Before是方法执行前调用

@After是方法执行后调用

@AfterReturning方法执行返回值调用。

 

Service层本身就可以切入事务，所以我们这类搞个常用的切controller层方法，每个执行controller层的方法都记录下请求Url，访问者IP 执行类方法参数等信息。

 

接着上面的环境

 

RequestAspect.java

package com.jluzh.aspect;

 

import javax.servlet.http.HttpServletRequest;

 

import org.aspectj.lang.JoinPoint;

import org.aspectj.lang.annotation.After;

import org.aspectj.lang.annotation.AfterReturning;

import org.aspectj.lang.annotation.Aspect;

import org.aspectj.lang.annotation.Before;

import org.aspectj.lang.annotation.Pointcut;

import org.jboss.logging.Logger;

import org.springframework.stereotype.Component;

import org.springframework.web.context.request.RequestContextHolder;

import org.springframework.web.context.request.ServletRequestAttributes;

 

@Component // 给Sprig管理

@Aspect // 切面

public class RequestAspect {

 

     private static Logger log = Logger.getLogger(RequestAspect.class);

    

     /**

      * 定义切面。log这个方法是不会被调用的，仅仅是切面定义的载体

      * public * 表示public方法，这个*表示任意返回值；.*.*(..) 第一个*表示controller下的任意类，第二个*表示任意方法，..表示任意参数。

      * 给符合这个表达式的方法都做切面，只要执行controller下的方式即可触发。

      */

     @Pointcut("execution(public * com.jluzh.controller.*.*(..))")

     public void log() {

     }

    

     /**

      * 方法执行前调用。注意方法不是指log方法，而是controller下的方法

      * @param joinPoint

      */

     @Before("log()")

     public void doBefore(JoinPoint joinPoint) {

         log.info("方法执行前。。。。");

         // 获取request的属性

         ServletRequestAttributes sra = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();

         // 请求的ip等

         HttpServletRequest request = sra.getRequest();

         log.info("uri: " + request.getRequestURI());

         log.info("ip: " + request.getRemoteHost());

         log.info("请求的方式: " + request.getMethod());

         log.info("方法全名: " + joinPoint.getSignature().getDeclaringTypeName() + "." + joinPoint.getSignature().getName());

         log.info("参数: " + joinPoint.getArgs()); // 参数

        

     }

    

     /**

      * 方法执行后调用。注意方法不是指log方法，而是controller下的方法

      * @param joinPoint

      */

     @After("log()")

     public void doAfter(JoinPoint joinPoint) {

         log.info("方法执行后。。。。");

     }

    

     /**

      * 方法返回时调用。注意方法不是指log方法，而是controller下的方法

      * @param result 被切面的方法的返回值

      */

     @AfterReturning(returning="result", pointcut="log()") 

     public void doAfterReturning(Object result) {

         log.info("方法返回值: " + result);

     }

}
